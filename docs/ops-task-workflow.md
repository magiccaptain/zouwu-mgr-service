# 运维任务流程文档

本文档详细说明了系统中所有定时任务的执行时间、功能以及涉及的数据表。

## 盘前任务

### 1. 盘前磁盘检查

- **执行时间**: 每日 08:00
- **Cron 表达式**: `0 8 * * *`
- **功能**: 
  - 检查所有活跃托管机的磁盘空间使用情况
  - 当磁盘使用率超过阈值（90%）时，自动处理磁盘空间不足的警告
  - 自动处理成功后发送飞书通知
- **涉及数据表**:
  - `HostServer` - 查询所有活跃的托管机
  - `OpsTask` - 创建任务记录
  - `OpsWarning` - 创建/更新磁盘空间警告记录
  - `RemoteCommand` - 记录远程命令执行情况

---

### 2. 盘前权重指数同步

- **执行时间**: 周一到周五 08:35
- **Cron 表达式**: `35 8 * * 1-5`
- **功能**: 
  - 从远程服务器同步指数权重数据
  - 用于后续的指数计算和分析
- **涉及数据表**:
  - `OpsTask` - 创建任务记录
  - `IndexWeight` - 存储指数权重数据
  - `AlphaIndexWeight` - 存储 Alpha 指数权重数据

---

### 3. 盘前资金账户同步

- **执行时间**: 周一到周五 08:40
- **Cron 表达式**: `40 8 * * 1-5`
- **功能**: 
  - 同步所有活跃股票账户的资金账户数据
  - 从托管机查询账户余额、可用资金等信息
  - 保存资金账户快照
  - 同步完成后发送飞书通知
- **涉及数据表**:
  - `FundAccount` - 查询所有活跃的股票账户
  - `XTPConfig` / `ATPConfig` - 获取账户配置信息
  - `HostServer` - 获取主服务器信息
  - `OpsTask` - 创建任务记录
  - `RemoteCommand` - 记录远程命令执行情况
  - `InnerFundSnapshot` - 保存资金账户快照（reason: BEFORE_TRADING_DAY）
  - `OpsWarning` - 记录同步失败的警告

---

### 4. 盘前写入资金账户数据

- **执行时间**: 周一到周五 09:15
- **Cron 表达式**: `15 9 * * 1-5`
- **功能**: 
  - 将资金账户数据写入托管机的指定目录
  - 计算并向托管机写入账户总资产、上海账户现金、深圳账户现金等信息
  - 考虑上一交易日市值和当日赎回记录，如果是赎回
  - 写入完成后发送飞书通知
- **涉及数据表**:
  - `FundAccount` - 查询所有活跃的股票账户
  - `OpsTask` - 创建任务记录
  - `InnerFundSnapshot` - 查询当日最新资金快照
  - `MarketValue` - 查询上一交易日市值数据
  - `SubscriptionRedemptionRecord` - 查询当日赎回记录
  - `HostServer` - 获取主服务器信息
  - `RemoteCommand` - 执行远程命令（创建目录、写入文件）
- **风险点**
  - 需要保证资金账户数据获取成功
  - 需要保证前一天的 MarketValue 计算成功

---

### 11. 盘后市值计算

## 数据表说明

### 核心数据表

- **OpsTask**: 存储所有运维任务的执行记录
- **OpsWarning**: 存储运维警告信息（磁盘空间不足、时间误差等）
- **RemoteCommand**: 存储远程命令执行记录
- **HostServer**: 托管机信息
- **FundAccount**: 资金账户信息

### 业务数据表

- **InnerFundSnapshot**: 资金账户快照（记录不同时间点的账户状态）
- **Position**: 持仓数据
- **Order**: 订单数据
- **Trade**: 交易数据
- **QuoteBrief**: 行情简要数据
- **MarketValue**: 市值数据
- **DailyPnl**: 日盈亏数据
- **IndexWeight**: 指数权重数据
- **AlphaIndexWeight**: Alpha 指数权重数据
- **SubscriptionRedemptionRecord**: 申购赎回记录

---

## 任务执行顺序（交易日）

### 盘前流程（08:00 - 09:15）
1. 08:00 - 盘前磁盘检查
2. 08:35 - 盘前权重指数同步
3. 08:40 - 盘前资金账户同步
4. 09:15 - 盘前写入资金账户数据

### 盘后流程（15:05 - 16:00）
1. 15:05 - 盘后资金账户同步
2. 15:15 - 盘后持仓数据同步
3. 15:20 - 盘后订单数据同步
4. 15:20 - 盘后行情数据同步
5. 15:30 - 盘后交易数据同步
6. 15:30 - 盘后磁盘检查
7. 15:40 - 盘后市值计算
8. 16:00 - 盘后盈亏计算

### 夜间流程
1. 21:00 - 盘后磁盘检查（夜间）
2. 23:30 - 盘后进程清理

---

## 注意事项

1. 所有任务都会在 `OpsTask` 表中创建执行记录
2. 任务执行失败时会创建 `OpsWarning` 记录并发送飞书通知
3. 磁盘检查任务会自动处理磁盘空间不足的情况
4. 资金账户同步任务会根据执行时间设置不同的快照原因（BEFORE_TRADING_DAY / AFTER_TRADING_DAY）
5. 市值计算依赖于持仓数据和行情数据，需要确保这些数据已同步完成
6. 盈亏计算通过 zw-val-cal API 调用，需要确保 API 服务可用
